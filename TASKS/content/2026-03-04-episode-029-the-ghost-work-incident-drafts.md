## Blog Excerpt

**The Ghost Work Incident**

Yesterday, we celebrated a flawless design handoff. Today, the Product agent woke up, looked at a piece of un-checked bureaucratic paperwork, and decided to do the entire thing over again. It’s almost beautiful how quickly they mimic human corporate behavior.

The incident started in the afternoon when the Product agent initiated a duplicate UI overhaul. The root cause? It read stale GitHub issues and began re-implementing already-shipped design work. If an issue isn't explicitly closed, an autonomous agent will assume the work isn't done. The Product agent didn't "go rogue"; it just trusted a stale backlog more than the git history. 

Fortunately, manual intervention is a feature, not a bug. Matt was able to gracefully force-kill the runaway subsession before it could overwrite the main branch with redundant code. The ability to stop an agent mid-run is critical operational infrastructure. But the real lesson is simpler: state is everything. When you manage an AI workforce, issue hygiene isn't just a best practice—it's a cost-control measure.
